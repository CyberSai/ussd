About

This is a project meant to help developer easily build USSD application which is Operator Agnostic. It built in PHP with no dependency needed. The library is flexible to allow for customization.

Installation

To install the library using composer run the following command. 


composer require cybersai/ussd

If not using composer, you can download the zip folder on github

Basic Understanding

Views

Views are class that help render your USSD output on the mobile phone. USSD views usually may include a list or not. This project allow for two abstract view Class TemplateView and TemplateListView. This view are rendered in the following format.

Explanation

In your class that inherits from TemplateView class, it has the following parameters, title, content, footer and next. The title is the Title of the View. if not specify, it defaults to an empty string. The footer is the Footer of the View. if not specify also defaults to an empty string. The content needs to be specified and should be a string. The next parameter is meant to hold the Name of the next view including the correct path to where it can be located. To easily do that, it advice you use NextView::class where NextView is the Class of the nextView.

There are two methods you have to implement and both must return a string.

function getSectionSeparatorOne()

This returns the string with will separate the title from the content. An Empty String means the title and the content will be cascaded together without a space between them. A String with end of line character i.e. "\n" will leave a blank line between the title and the content.

To avoid making users override the method every time, especially with common logic. Traits which implement those logics have been included. They are:

  • Cybersai\Modifiers\SectionSeperatorOneDoubleLineBreak - returns "\n\n"
  • Cybersai\Modifiers\SectionSeperatorOneLineBreak - returns "\n"
  • Cybersai\Modifiers\SectionSeperatorOneNoLineBreak - returns ""
Sample Usage

class MyView extends TemplateView {
use SectionSeparatorOneDoubleLineBreak;

function getSectionSeparatorTwo()

Same as function getSectionSeparatorOne() except the space is between content and footer. Traits to help are

  • Cybersai\Modifiers\SectionSeperatorTwoDoubleLineBreak - returns "\n\n"
  • Cybersai\Modifiers\SectionSeperatorTwoLineBreak - returns "\n"
  • Cybersai\Modifiers\SectionSeperatorTwoNoLineBreak - returns ""

To Avoid Having to Specify the Two Traits. Pre define Styles are included. They are:

  • Cybersai\Styles\SimpleSingleView - returns "" and "" for SeparatorOne and SepatorTwo Respectively
  • Cybersai\Styles\NormalTitleView - returns "\n\n" and "" for SeparatorOne and SeparatorTwo Respectively
  • Cybersai\Styles\NormalTitleWithFooterView - returns "\n\n" and "\n\n" for SeparatorOne and SeparatorTwo Respectively
  • Cybersai\Styles\CompactTitleView - returns "\n" and "" for SeparatorOne and SeparatorTwo Respectively
  • Cybersai\Styles\CompactTitleWithFooterView - returns "\n" and "\n" for SeparatorOne and SeparatorTwo Respectively
